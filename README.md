## 🔐 Authentication Flow

### 1. Login Page (`/auth/login`)
- Username/Password form
- Calls `POST /auth/login`
- Receives JWT token
- Stored in localStorage and React Context
- Redirects to Dashboard

### 2. Signup Page (`/auth/signup`)
- Captures device `info from URL params (MAC, IP, router)
- Creates user account
- Auto-login and redirects to Dashboard
- User can immediately view available plans

### Context Usage
```typescript
const { user, login, logout, macAddress, routerIdentity } = useAuth();

user: { id, username, isActive, sessionExpiry }
isAuthenticated: boolean
isLoading: boolean
macAddress: string (from device)
routerIdentity: string (router name)
```

## 💳 Payment System

### Dashboard Payment Flow

**1. Bundle Purchase Page**
```
Location: Dashboard → Plans Tab
- Display all available plans
- Show price and duration
- User selects plan and quantity
```

**2. Payment Initiation**
```typescript
const response = await apiFetchPost('/payments/initiate', {
  planId,
  phone: phoneNumber,
  email: autoGeneratedEmail,
  macAddress,
  userIp,
  routerIdentity,
  password,
  isGift: false,
  externalId: Date.now().toString()
});
```

**3. PaymentStatusMonitor Automatically Starts**
```
- Component mounts with transactionId
- Polls /payments/status/{transId} every 3 seconds
- Shows loading screen: "Processing Payment..."
- Max 40 attempts (2 minutes total)
```

**4. Payment Confirmation**
- When status = "SUCCESSFUL"
- Stores metadata in localStorage:
  - `wifiSessionUsername`
  - `wifiSessionDuration`
  - `wifiPaymentIsGift`
  - `wifiPaymentRecipientUsername`
- **Redirects to `/payment-success` page**

**5. Success Page Display** 🆕
```
Components shown:
├─ Green checkmark icon
├─ "Payment Successful!" title
├─ Session details (username, duration)
├─ "What's Next?" instructions
├─ Auto-redirect countdown (5 seconds)
├─ "Go to WiFi Login Portal" button
└─ "Return to Dashboard" button
```

**6. Auto-Redirect**
- After 5 seconds → redirects to WiFi login portal
- OR user clicks button to go immediately
- Portal URL from localStorage:
  - `wifiLinkLogin` (primary)
  - `wifiLinkOrig` (fallback)
  - `/auth/login` (default)

**7. WiFi Login**
- User taken to login.html
- Logs in with credentials
- Gets internet access ✅

### Gift Purchase Flow

**1. Gift Tab in Dashboard**
```
Dashboard → Bundles Tab → Gift Option
├─ Recipient username field
├─ Plan selector
├─ Phone number (payer)
└─ Recipient password (manual or auto-generate)
```

**2. Gift Payment**
```typescript
const response = await apiFetchPost('/payments/initiate', {
  planId,
  phone: payerPhone,
  email: payerEmail,
  isGift: true,
  recipientUsername: 'Alice',
  password: 'AlicePass123',
  externalId: Date.now().toString()
});
```

**3. Show Credentials**
```
After successful payment:
┌─────────────────────────────┐
│ ✅ Gift Successfully Sent   │
│                             │
│ Recipient: alice            │
│ Username: alice             │
│ Password: AlicePass123      │
│ Duration: 24 hours         │
│                             │
│ Share these with recipient  │
└─────────────────────────────┘
```

**4. Recipient Login**
- Receives credentials from payer
- Opens WiFi login portal
- Logs in with provided credentials
- Gets 24 hours of internet access ✅

## 🔄 Data Flow Examples

### Complete Payment Flow
```
User inputs → Payment form
           ↓
API call POST /payments/initiate
           ↓
Response: { transId, status }
           ↓
PaymentStatusMonitor starts polling
           ↓
Polls GET /payments/status/{transId}
           ↓
Status = SUCCESSFUL
           ↓
localStorage updated with session info
           ↓
Redirect to /payment-success
           ↓
Success page displays
           ↓
Auto-redirect to login portal (5s)
           ↓
```

```
├─ Recipient username

└─ Recipient password

Backend creates/reactivates recipient automatically

Webhook confirms payment

Success shows instructions to share credentials
```

## 🐛 Common Issues

### Payment stuck on "Processing"
1. Check network connectivity
2. Verify API_URL is correct
3. Check backend is running
4. Try manually going to success page

### Token expiration
1. Logout and log back in
2. New JWT token issued
3. Session preserved in context

### WiFi portal not loading
1. Check `wifiLinkLogin` in localStorage
2. Verify router IP is accessible
3. Check WiFi connection
4. Try manual login at `/auth/login`

### Gift payment shows as self-purchase
1. Check `isGift` flag in payment request
2. Verify recipient username not empty
3. Check backend logs for gift indicator
